# 워치리스트 분류 기준 — 의료 방사선 중심

> 사람용 설명서. **기계용 단일 권위는 `watchlist.toml`**, 이 문서는 그 "왜"를 적는다.
> (2026-06-30 확정. 분류·tier 변경 시 둘을 함께 갱신.)

## 1. 스코프 — 의료 방사선 안전

대상 = **의료(핵의학·방사선종양·영상의학) 현장의 방사선 안전**. lawbot 이 답할 질의의 도메인이다.

> ⚠️ 경위: 한때 "방사선안전 *전반*"으로 넓혔다가 철회했다(2026-06-30, Dr. Ben). 생활방사선(라돈·
> 우주방사선·NORM)·원전 방재·산업·환경·수의·식품 방사선은 *의료 현장에 적용되지 않아* 스코프 밖.
> 핵심 교훈: **"법 전체가 방사선"과 "의료 방사선"은 다르다.** 전자만 보면 생활방사선법·방재법까지
> 끌려들어온다. 의료 적용 게이트가 경계를 닫는다.

## 2. 법령 위계 복습 — 부령 vs 고시 (분류의 전제)

```
법률 (Act, 국회)
 └ 시행령 (대통령령)
    └ 시행규칙 / 부령 (보건복지부령 등)  ← 여기까지 "법령"(법규명령, 구속력 ○)
       └ 행정규칙 (고시·훈령·예규)        ← 법규성 약함, 별도 트랙
```

- **진단용방사선발생장치의안전관리에관한규칙**, **특수의료장비의설치및운영에관한규칙** 은
  **고시가 아니라 보건복지부령(부령)** 이다. 의료법 제37·38조가 위임한 **독립 법령**으로,
  의료법 *자체* 시행규칙과는 별개의 부령이다(각자 법령ID 008391·009476).
- 따라서 이 둘은 **법령 watchlist** 에 속한다(고시 수집 트랙 아님). 그리고 부령이라 **자체
  법령ID로 독립 추적**되므로, 모법(의료법)과 분리해 별도 tier 로 둘 수 있다.

## 3. 분류 기준 — 두 게이트의 곱

| | ① 방사선 규율 정도 | ② 의료 적용 | → tier |
|---|---|---|---|
| 법/규칙 전체가 방사선안전 | whole | 적용 ○ | **core** |
| 모법은 일반, 일부 조만 방사선 | partial | 적용 ○ | **peripheral** |
| (어느 경우든) 의료에 적용 안 됨 | — | 적용 ✗ | **제외** |

- **core** = whole + 의료적용 → whole-law 감시 + 전문(全文) 코퍼스 수집.
- **peripheral** = partial + 의료적용 → 방사선 조만 감시(§6) · 코퍼스엔 의료법이라 포함 가능.
- **제외** = 의료 미적용. '법 전체가 방사선'이라도(생활방사선법·방재법) 의료 무관이면 제외.

**핵심 따름정리**: 모법이 일반(의료법)이라 peripheral 이어도, 그 위임 *방사선 전용 부령*
(진단용방사선규칙·특수의료장비규칙)은 부령 전체가 의료 방사선 → **독립 core**. 부령이라
자체 법령ID로 떼어낼 수 있어 가능한 분리다.

## 4. 현재 분류

**core (3)** — 전체가 방사선안전 + 의료 직접 적용
- 원자력안전법 — 핵의학·방종의 방사선안전 근거(RI 사용허가·선량한도·방사선작업종사자·방어).
  법의 상당부가 원전이나, 의료 RI 안전 조항이 법 전체에 흩어져 있어 조 단위 분리 불가 → whole-law core.
- 진단용방사선발생장치의안전관리에관한규칙 — 영상의학 진단 X선 전용(의료법 §37 위임).
- 특수의료장비의설치및운영에관한규칙 — CT·유방촬영 QA(의료법 §38 위임). MRI 비전리 일부 포함 감수.

**peripheral (4)** — 모법은 의료·의약 일반, 일부 조만 방사선
- 의료법(§37·38) · 약사법(방사성의약품 §54 등) · 의료기기법(방사선 의료기기) · 의료기사등에관한법률(방사선사).

**제외 (15)** — 의료 미적용(스코프 밖). law_id 는 `watchlist.toml` 끝 주석에 복구용 보존.
- 생활주변방사선법 · 방사능방재법 · 방진법 · 산업안전보건(기준규칙) · 연구실안전법 · 수의사법 ·
  먹는물 · 식품위생법 · 수입식품 · 실내공기질 · 학교보건 · 재난안전법 · 폐기물 · 환경정책기본법 ·
  비파괴검사법. (그 외 전파법[비전리]·관세법·간호법.)

## 5. core 와 peripheral 의 비대칭 (왜 core 는 쉽고 peripheral 은 어려운가)

- **core 는 쉽다**: "법/부령 전체가 방사선"이라 *전체를 통째로* 감시·수집하면 끝. 어느 조가 바뀌든 본다.
- **peripheral 은 어렵다**: 모법(의료법)은 방사선과 무관한 사유로 *자주* 개정된다. 통째로 감시하면
  무관 개정마다 STALE 노이즈. 그렇다고 "방사선 조"를 손으로 못박으면 — 개정 때 **조 재번호**·
  **새 방사선 조 신설**에 취약하고, 본문을 다 읽어야 한다.

## 6. peripheral 모니터링 설계 — keyword-at-diff-time

**조 번호를 미리 못박지 않는다.** 개정이 일어났을 때 판정한다:

```
peripheral 법 개정 감지 (MST 변경)
 → 직전 버전 ↔ 새 버전 diff → 바뀐 조 추출
 → 바뀐 조에 방사선 키워드(방사선·방사성·동위원소·피폭·선량·핵종·방사능) 있나?
      ├ 있음 → ⚠️ 검토 알림 (방사선 조가 실제로 바뀜)
      └ 없음 → 조용히 스냅샷 갱신 (무관 개정 = 무시)
```

- 강건: 조 재번호·신설에 무관(키워드가 diff 시점에 판정). 본문 선독 불필요.
- `watch_articles` = **사람용 힌트**(예상 조), 탐지의 키가 아니다 — 지금 완벽히 못박지 않아도 됨.

**구현 2단계**
- **Interim (구현됨)**: peripheral 도 MST 추적하되, 개정 시 STALE 이 아니라 **REVIEW nudge**
  ("주변법 X 개정 — 방사선 조 영향 확인 要 · watch: …"). 자동 키워드 판정 없이 Dr. Ben 이 수동 확인.
  → `_freshness_audit.py` 의 PERIPHERAL 섹션.
- **Full (후속)**: 위 keyword-at-diff-time 자동화(조 파싱+diff+키워드). legalize-kr `.md` 가
  `##### 제N조` 로 구조화돼 있어 조 단위 diff 가 가능.

## 7. 권위 관계

- `watchlist.toml` — 기계용 단일 권위(fetch·audit 가 읽음). tier·law_id·watch_articles.
- 이 문서 — 사람용 "왜"(분류 논거·설계). 변경 시 둘을 함께 갱신.
- `_watchlist.py` — 로더(tier 운반). `_freshness_audit.py` — core STALE + peripheral REVIEW.

## 8. RAG 코퍼스 설계 (미래) — 다운로드 7 전부 / 인덱싱은 tier 별

**다운로드(원문 확보)와 RAG 인덱싱(무엇을 벡터화)을 분리한다.**

- **다운로드 = 7 family 전부, tier 무관** (`update_laws.sh --all`). 원문은 다 갖춘다.
- **RAG 인덱싱 단위는 tier 별로 다르다**:
  - **core 3** → **전문(全文)** 청킹. 법/부령 전체가 의료 방사선이라 맥락 보존(딥리서치 권장 = 관계법령 전체 확보).
  - **peripheral 4** → **방사선 조만** 청킹(+상위 조·법령명 prefix). 의료법 전체(의료광고·개설요건 등)를
    넣으면 검색 오염·스코프 붕괴 → 방사선 조(의료법 §37·38, 약사법 §54 등)만 인덱싱.

**왜**: 법률 RAG 는 오검색 = 틀린 법적 답변. peripheral 통째 인덱싱은 lawbot 을 스코프 밖
(의료기관 개설 등)으로 끌고 가고 방사선 질의 정밀도를 떨어뜨린다.

**한 메커니즘 두 용도**: peripheral **방사선 조 추출**이 §6 모니터링(개정된 조에 방사선?)과
*동일 작업*이다. Full 단계의 조 파싱+키워드 도구 하나가 모니터링·코퍼스 둘 다를 굴린다.

```
7 family 다운로드
 ├ core 3      → 전문 → RAG
 └ peripheral 4 → [방사선 조 추출] → RAG   (이 추출 = monitoring REVIEW 와 공유)
```

**v1 점진 경로(옵션)**: 빠른 출시엔 peripheral 도 일단 통째 넣고 rerank 에 맡긴 뒤, 조 필터로
정밀화 — 정밀도와 출시속도의 트레이드오프. 정밀 우선이면 처음부터 조 필터.

## 9. 고시(행정규칙) watchlist화 — 명시 큐레이션 (키워드 선별 폐기)

법령(법률·부령)은 자체 법령ID로 깔끔히 목록화되지만, 고시 *선별*은 다르다.
**키워드 sweep 이 동음이의로 망가진다**:
- `방호` = 방사선 防護 ∩ 경비 防護(방호원·청사방호·소방호스)
- `방재` = 방사능 防災 ∩ 재난 防災(방재기상·도로터널) ∩ 지"방재"정(부분일치)

실측 95건 중 노이즈 다수 → **선별 로직은 키워드가 아니라 명시 큐레이션 목록**
(`watchlist.toml` `[[admrule]]`). 키워드는 후보 *발견(discovery seed)* 에만 보조,
최종 선별은 사람이 확정한 목록이다.

- **현재(의료 중심) 17건**: 질병청 진단용 고시 2 + 원안위 핵의학·방종 고시 13 + 방사선비상진료 2.
  각 항목 `parent` 로 위임 근거 core 법에 매핑.
- `_collect_admrul.py` 가 이 목록을 읽어 admrule-kr 에서 본문.md 수집(키워드 sweep 폐기).
  fetch 경로 = `<agency>/<kind>/<name>/본문.md` (IP 무관·CI 가능).

> ⚠️ 혼동 주의: 키워드를 쓰는 건 §6 peripheral *법* 모니터링(이미 선별된 법의 *개정된 조*에
> 방사선 키워드 있나)이지, 고시 *선별*이 아니다. 선별=명시목록 / 변경감지=키워드.

## 부록 — 고시 RAG 인덱싱 (미래)

고시는 모두 위임 근거가 방사선 전용(의료분야 기술기준 등)이라 **전문(全文) 인덱싱**이 기본
(core 법과 동급). 별표(선량계수·QA 주기 등 정량표)는 메타데이터 태깅해 정밀 검색 대응(딥리서치 §3 청킹).
