# law.go.kr (국가법령정보센터) DRF OpenAPI — 요청 파라미터 레퍼런스

`_freshness_audit.py`(신선도 감사) 가 호출하는 **law.go.kr DRF OpenAPI** 의 요청 인자·회신 필드 정리.
"내가 보내는 조건(파라미터)" 과 "돌아오는 데이터(필드)" 를 구분하는 게 이 문서의 핵심.

- 엔드포인트: `https://www.law.go.kr/DRF/lawSearch.do` (목록 검색) · `lawService.do` (본문 조회)
- 인증: `OC=` 파라미터 = 가입 이메일 @앞 (이 프로젝트는 `benkorea.ai`).
- ⚠️ **IP 등록 필요**: 직접 호출이라 호출 PC 공인 IP 가 open.law.go.kr 에 등록돼야 함. 단 **이 PC 는 테스트용 등록 완료**, WSL2·Windows 가 같은 공인 IP 공유 → 양쪽 Claude 모두 호출 가능. (CI/GitHub Actions 는 IP 가변이라 불가 — 그래서 데이터 수집은 legalize-kr 미러 경유.)

## 핵심 구분 — 요청 인자 ≠ 회신 데이터

```
[ 내가 보내는 요청 인자 ]                    [ 돌아오는 회신 데이터 ]
 target  = 어떤 데이터셋을 줄까 (eflaw 등)     법령 한 건당 아래 12개 필드 세트
 query   = 법령명에 이 단어 (이름 1필드만!)      (필드 선별 불가 — 통째로 옴)
 ancYd   = 공포일자 이 범위
 org     = 소관부처 이 코드
 ...                                          → 원하는 필드는 받아서 코드로 추출
```

**중요**: `query` 는 만능 필터가 아니라 **법령명(이름) 한 필드 전용** 검색어다. 다른 필드로 거르려면
그 필드 **전용 파라미터**를 따로 붙인다(AND 결합). 전용 파라미터가 없는 필드(`현행연혁코드`·MST 등)는
서버측 필터 불가 → 받아서 클라이언트에서 거른다.

## 1. `lawSearch.do` 요청 파라미터

검증: ✓ = 이 프로젝트에서 라이브 호출로 동작 확인(2026-06-30) / ○ = 공식문서 기준, 미검증.

| 파라미터 | 담당 필드 / 역할 | 예시 값 | 검증 |
|---|---|---|---|
| `OC` | 인증(가입 이메일 @앞) | `benkorea.ai` | ✓ |
| `target` | **데이터셋 종류** — `law`(현행)·`eflaw`(시행일법령, 시행예정 포함)·`admrul`(행정규칙) | `eflaw` | ✓ |
| `type` | 응답 포맷 | `XML` / `JSON` / `HTML` | ✓ (XML) |
| `query` | **법령명**에 포함된 단어 (이름 전용) | `방사선` | ✓ |
| `display` | 페이지당 건수(최대 100) | `100` | ✓ |
| `page` | 페이지 번호 (`totalCnt`/`display` 로 총 페이지 산출) | `1` | ✓ |
| `ancYd` | **공포일자 범위** `YYYYMMDD~YYYYMMDD` | `20260101~20260630` | ✓ |
| `efYd` | **시행일자 범위** `YYYYMMDD~YYYYMMDD` | `20270101~20271231` | ✓ |
| `org` | **소관부처 코드** | `1079960`(원자력안전위원회) | ✓ |
| `nb` | 공포번호 (전역 고유 아님 — 서로 다른 법이 같은 번호 가능) | `21679` | ✓ |
| `rrClsCd` | 제개정구분 코드 — **`300202`=일부개정**(확인). 제정·전부개정 등은 미확인 | `300202` | ✓ (300202만) |
| `sort` | 정렬 — `ddes`(공포일자 내림)·`dasc`(공포일자 오름)·`efdes`/`efasc`(시행일자)·`lasc`/`ldes`(법령명) | `ddes` | ✓ |

> `query` 없이 `ancYd`·`org` 만으로도 **전체 corpus 에서** 날짜·부처로 거를 수 있다(라이브 확인 — query 없이 B·C 동작). 즉 "이 부처 소관 + 이 기간 공포분 전부" 를 1콜로 받을 수 있다.

## 2. `lawService.do` 요청 파라미터 (본문 조회)

| 파라미터 | 역할 | 비고 |
|---|---|---|
| `OC`·`target`·`type` | 위와 동일 | — |
| `MST` | **법령일련번호**(버전별 고유) 로 본문 지정 | `_freshness_audit.py` 가 MST 존재확인에 사용 |
| `ID` | **법령ID** 로 지정(MST 와 혼동주의 — `ID=` 는 법령ID 를 기대) | 함정: 메모리 `reference_moleg_openapi_gotchas` |
| `JO` | 특정 **조(條)** 만 조회 (예: `000200`=제2조) | 본문 일부만 받기 |

## 4. radsafety 일일 "새 개정 감지" 함의

- `target=eflaw` 필수 — `target=law`(현행)는 시행 중 버전만 줘서 **공포됐으나 미시행(시행예정)** 개정을 놓침.
- 판단축은 **`공포일자`**(개정 사건 시점), `시행일자` 아님. 견고한 신호는 **저장 MST ↔ API 최신 공포 MST 대조**(달력 today 필터보다 미러 지연·하루 누락에 내성).
- **부처 그물의 한계**: radsafety 7패밀리가 **3개 부처**(원안위 `1079960` / 보건복지부 / 식약처)에 걸쳐 있어 `org` 하나로 다 못 덮음. → `ancYd`(어제~오늘) × 부처별 그물로 넓게 + watchlist MST 대조로 정밀, 하이브리드 권장.
